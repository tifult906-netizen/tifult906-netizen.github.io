- **All 4 sections** have `role="region" aria-labelledby`. ✓

- **Main** landmark present. ✓
- **Lang bar** uses `<div role="navigation">` instead of native `<nav>`. **FIXED** — changed to `<nav>`.
- **Consent checkbox** is wrapped in `<label>` (implicit association) but has no `aria-describedby` linking to `err_consent`. Error message added to `aria-describedby`. **FIXED.**
- **Stat band** `<div>` has no role/landmark — added `role="region" aria-label`. **FIXED.**
- **Form element** has no `aria-label` or `aria-labelledby`. Added `aria-labelledby="form-title"`. **FIXED.**
- **Ticker images in duplicate set** have `alt` text — but the duplicate wrapper correctly uses `aria-hidden="true"`. ✓
- **Color contrast:**

  - ink on sun (#F5B82E): **10.27:1** ✓
  - ink-muted on bg-paper-deep: **4.79:1** ✓

#### Tech Debt
- `left: var(--space-3)` and `left: var(--space-5)` used throughout for the accessibility widget and logo. These are physical properties; in RTL the accessibility widget should be on the LEFT (start side), which happens to be physical left. Correct for RTL, but fragile for LTR switch — minor, left as-is (widget position is intentionally fixed bottom-left regardless of language).
- Steps use physical `padding-right` overrides per child — would benefit from logical properties but are non-trivial to change without visual regression.

## 2. Strategy

### Priority Order (highest impact first)
1. **A11y: skip links** on secondary pages (IS 5568 hard requirement for public Israeli site)
2. **A11y: form ARIA** (consent aria-describedby, form aria-labelledby)
3. **A11y: native `<nav>`** everywhere (removes redundant role=navigation)
4. **A11y: focus-visible** on secondary pages
5. **SEO: meta description + canonical** on secondary pages
6. **RTL: list margin-inline-start** for bidirectional correctness
7. **RTL: back-link arrow direction** (← → → in RTL)
8. **Landmarks:** stat-band region, form aria-labelledby, header role=banner
9. **Performance:** img lazy/decoding/dimensions on ticker logos
10. **Minor:** move skip-link target to `<main>` element itself

### What NOT to change
- Form field names, webhook URL, validation logic — per CLAUDE.md constraint
- All Hebrew copy — exact preservation
- Visual design, layout, color tokens
- Animation logic
- Structured data in index.html
- External link rel attributes (already correct)
